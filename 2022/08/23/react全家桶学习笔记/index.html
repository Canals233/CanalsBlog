    虚拟DOM

    虚拟DOM是真实DOM的复制，真实DOM在浏览器改变很慢，在JS中的虚拟会快很多

    第一个虚拟DOM将作为快照，第二个是第一个虚拟DOM的复制

    在虚拟复制DOM进行变化，变化结束后，将快照和变化树进行比较，之后真实DOM回流到和快照一样

    redux-thunk

    处理异步的操作,dispatch参数是对象

    主要意思就是声明一个函数来代替表达式，这样就可以将执行求值操作（evaluation）延迟到所需要的时刻。

    redux-thunk确实就是干这个的。它就是个中间件，去监控传入系统中的每一个action，如果是个函数的话，那么它就会调用那个函数。这就是redux-thunk的职责, 提供一个dispatch异步操作的环境。

    Redux会传递两个参数给thunk函数：dispatch - 如果需要的话可以 dispatch 新的 action;getState - 用于访问当前 state。getState还可用于决定是获取新数据还是返回缓存结果，这取决于当前 state。 这就是redux-thunk做的事情了。

    saga自我总结

    1.主要是通过监听reducer活动进行，因此需要创建监听器和监听器组，这些监听器组将在激活的saga中间件中使用，通常用takelatest函数进行节流

    2.在这些实践中，all只在监听组中使用

    3.监听器触发的功能函数中call用的多，call在需要调用异步函数或者创建标准对象时候使用，call会执行括号里函数并运行所有异步操作。

    4，put就是dispatch，里面接收一个标准对象

    性能优化之

    但只在确实需要性能优化的时候去用！

    • 变量声明和匿名函数都会在每次函数被触发后，不过问题不是很大
    • ucb相当于一个缓存机制
    • usecallback（要缓存的数据或函数,[依赖数组]）,如果内部的数据不在依赖里面，里面的值将不会更新
    • const val= useMemo（要缓存的函数，依赖数组）
      • 只在第一次计算时候进行实际计算，将return的结果之保存，以后函数重调用不会进入内部的缓存函数，除非依赖变化
    • memo（）函数，memo包裹一个组件，类似购物车，原来新添加的仍然会触发购物车内的其他物品卡片渲染，使用memo包裹购物车组件后，原来添加的物品卡片就不会再渲染
    • 不要让主页一进来就下载所有东西，进行分包
      • 正常引入是静态引入，逐行进行，成功引入后才会继续往下
      • 动态引入lazy，Suspenseconst Home=await import("./routers/home/home") js的基础形式
      • React中使用lazy const Home=lazy(()=>import("./routers/home/home"))这将得到一个lazy组件，在需要时才实际渲染,但要包裹在<Suspense fallback={<Spinner/>}中才有用，fallback是没加载的时候的替代显示
      • 这样实际上就进行了一个分包，使组件在需要的时候才被加载

    React开发原则

    1.不要过于复杂，应在运行效率和代码简单程度上合理分配

    2.不要追新，等到最佳实现和代码整体稳定再用

    3.代码分割的原则是组合，而不是把一个原来的大项目强行打散为小项目

    跨端应用

    即PWA，PWA使web应用有了原生应用的优点

    web应用的基础是网络，而原生应用的基础是本地能离线运行

    PWA应用就是结合了两点，能离线运行部分功能，也能调用手机硬件的原生功能，这主要是HTML5给的API

    有趣的是，有固定生态的苹果不太接收PWA,而谷歌等大型网络公司很喜欢PWA

    开发原则

    • 使用HTTPS来加强网络安全，类似lets enctry和cloudflare都免费

    • Web 应用程序清单 web-manifest 的目的是将 Web 应用程序安装到设备的主屏幕，为用户提供更快的访问和更丰富的体验。 类似于基础设置，相当于微信小程序的json设置，比如背景，icons之类

    • <meta name="viewport" content="width=device-width, initial-scale=1, shrink-to-fit=no">这一行很重要，保证了你的程序能在不同屏幕缩放

    • service worker，一个服务器与浏览器之间的中间人角色，如果网站中注册了service worker那么它可以拦截当前网站所有的请求，进行判断（需要编写相应的判断程序），如果需要向服务器发起请求的就转给服务器，如果可以直接使用缓存的就直接返回缓存不再转给服务器。从而大大提高浏览体验。
